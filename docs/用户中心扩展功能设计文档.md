## 一、需求概述

### 1.1 功能需求

用户在个人中心可以：
1. 👥 查看"我的关注"列表 - 我关注的所有用户
2. 👫 查看"我的粉丝"列表 - 关注我的所有用户
3. 👤 访问其他用户主页 - 查看他人的信息、发布等

### 1.2 页面清单

| 页面 | 路径 | 描述 |
|------|------|------|
| 关注列表 | `/pages/follows/index` | 展示当前用户关注的所有人 |
| 粉丝列表 | `/pages/followers/index` | 展示关注当前用户的所有人 |
| 用户主页 | `/pages/user-detail/index` | 展示某个用户的完整信息 |

### 3.3 与关注列表的区别

| 特性 | 关注列表 | 粉丝列表 |
|------|---------|---------|
| 显示对象 | 我关注的人 | 关注我的人 |
| 按钮 | "已关注" + 取消按钮 | "+ 关注" 或 "已关注" |
| 互粉标签 | 无 | 有（如互相关注） |
| 查询条件 | 当前用户的 follows | followingId = 当前用户 |

### 3.4 互粉逻辑

```javascript
// 检查是否互粉
async function checkMutualFollow(userId) {
  // 1. 检查我是否关注了这个用户
  const iFollowHim = await followService.isFollowing(userId)
  
  // 2. 检查这个用户是否关注了我
  const heFollowsMe = await checkIfUserFollowsMe(userId)
  
  // 3. 如果都是，则显示 "互粉✓" 标签
  return iFollowHim && heFollowsMe
}
```

## 五、数据流和 API 调用

### 5.1 关注列表页面

```
onLoad(options)
  └─ getFollowingList(page=1, pageSize=20)
     └─ followService.getFollowingList()
        └─ db.collection('follows').get()
           返回当前用户的所有 follows 记录

onReachBottom()
  └─ getFollowingList(page++, pageSize=20)
     └─ 加载下一页数据

取消关注(userId)
  └─ followService.unfollow(userId)
     └─ db.collection('follows').where({ followingId: userId }).remove()
```

### 5.2 粉丝列表页面

```
onLoad(options)
  └─ getFollowerList(page=1, pageSize=20)
     └─ db.collection('follows').where({ followingId: currentUserId }).get()
        返回所有 followingId === 当前用户的记录

关注粉丝(userId)
  └─ followService.follow(userId)
     └─ db.collection('follows').add({ followingId: userId })
```

### 5.3 用户主页

```
onLoad(options)
  ├─ 如果 userId 为空，认为是自己的主页
  ├─ 调用 authModel.getUserInfo(userId)
  │  └─ 获取用户基本信息
  ├─ 调用 followService.isFollowing(userId)
  │  └─ 检查是否已关注
  ├─ 调用 postService.getUserPosts(userId)
  │  └─ 获取用户发布的内容
  └─ 将数据显示在页面上

关注该用户
  └─ followService.follow(userId)

点击粉丝数
  └─ 跳转到 /pages/followers-detail/index?userId=xxx

点击关注数
  └─ 跳转到 /pages/following-detail/index?userId=xxx
```

## 九、数据库设计回顾

这三个页面都基于 `follows` 集合：

```
集合名: follows

权限: 仅创建者可读写

字段：
- _id: UUID（系统生成）
- _openid: 关注者 ID（系统自动添加）
- followingId: 被关注者 ID
- followingData: 被关注者信息快照
- createTime: 创建时间（系统自动）

关键查询：
- 获取我的关注列表：自动按 _openid 过滤
- 获取我的粉丝列表：where({ followingId: 我的 openid })
- 获取他人的粉丝列表：where({ followingId: 他人的 openid })
- 获取他人的关注列表：需要在他人的 follows 权限允许时查询
  （当前权限设置为"仅创建者可读写"，所以只能查看自己的）
```

## 十、样式统一性指南

### 10.1 颜色和字体

- 主色调（蓝色）：用于高亮操作按钮
- 次色调（灰色）：用于禁用状态或次要信息
- 文字大小：保持与个人中心一致
- 圆角：16rpx 卡片，50% 头像

### 10.2 操作反馈

- 关注成功：提示 "已关注"
- 取消成功：提示 "已取消关注"
- 加载中：显示骨架屏或加载动画
- 错误：显示错误提示和重试按钮

### 10.3 空状态

```
图片：
  关注列表空：心形 + "还没有关注任何人"
  粉丝列表空：人形 + "还没有粉丝"
  
文字：
  "还没有关注任何人，去发现更多用户"
  
按钮：
  "去发现" → 跳转到发现/首页
```
