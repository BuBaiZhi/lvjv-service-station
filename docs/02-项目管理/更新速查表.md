# 更新速查表

## 🎯 核心答案

**不需要每次都全部复制！只复制修改过的文件。**

---

## 📋 更新流程

### 第一次（初始化）
```
1. 复制所有文件到 HBuilderX 项目
2. npm install pinia uni-ui
3. 运行项目
```

### 后续每次更新
```
1. 修改源项目中的文件
2. 只复制修改过的文件到 HBuilderX 项目
3. 保存文件（Ctrl + S）
4. 浏览器自动刷新
```

---

## 📁 哪些文件需要复制？

| 修改内容 | 需要复制 | 例子 |
|---------|--------|------|
| 修改组件 | ✅ 是 | UserCard.vue |
| 修改样式 | ✅ 是 | variables.scss |
| 修改逻辑 | ✅ 是 | user.ts |
| 添加新文件 | ✅ 是 | NewComponent.vue |
| 修改文档 | ❌ 否 | README.md |
| node_modules | ❌ 否 | 不需要 |

---

## 🔧 使用 Git 管理

### 初始化
```bash
git init
git add .
git commit -m "初始项目"
```

### 查看修改
```bash
git status          # 查看修改了哪些文件
git diff            # 查看具体修改内容
```

### 提交更新
```bash
git add .
git commit -m "更新说明"
```

---

## ⚡ 快速更新步骤

```
1. 在源项目中修改文件
2. 复制修改的文件到 HBuilderX 项目
3. 保存（Ctrl + S）
4. 浏览器自动刷新（1 秒）
5. 查看效果
```

---

## 💡 最佳实践

✅ **推荐**：
- 使用 Git 管理源代码
- 只复制修改的文件
- 定期提交更新

❌ **不推荐**：
- 每次都复制所有文件
- 不使用版本控制
- 手动跟踪修改

---

## 🎉 总结

**答案**：不需要全部复制，只复制修改过的文件！

**时间**：修改后 1 秒看到效果

**工具**：使用 Git 管理版本
