# 项目更新指南

## 🎯 核心原则

**不需要每次都全部复制！** 只需要复制**修改过的文件**。

---

## 📁 推荐的工作流程

### 方案 A：使用 Git 版本控制（推荐）⭐

#### 第一次设置

```bash
# 1. 在项目目录初始化 Git
git init

# 2. 添加所有文件
git add .

# 3. 提交初始版本
git commit -m "初始项目框架"
```

#### 后续更新

```bash
# 1. 查看修改了哪些文件
git status

# 2. 添加修改的文件
git add .

# 3. 提交更新
git commit -m "更新说明"

# 4. 查看更新历史
git log
```

**优点**：
- ✅ 清楚地看到哪些文件被修改了
- ✅ 可以回滚到之前的版本
- ✅ 便于团队协作
- ✅ 完整的版本历史

---

### 方案 B：手动管理（简单）

#### 第一次

```
复制所有文件到 HBuilderX 项目中
```

#### 后续更新

只复制**修改过的文件**到 HBuilderX 项目中。

**例如**：
- 修改了 `components/UserCard.vue`
- 只需复制这一个文件到 HBuilderX 项目中
- 其他文件不需要动

---

## 🔄 具体的更新流程

### 场景 1：修改了一个组件

**例如**：修改了 `components/UserCard.vue`

```
1. 在源项目中修改文件
2. 保存文件
3. 复制修改后的文件到 HBuilderX 项目中
4. HBuilderX 会自动刷新
5. 浏览器自动显示最新效果
```

### 场景 2：修改了多个文件

**例如**：修改了 `components/UserCard.vue` 和 `styles/variables.scss`

```
1. 在源项目中修改文件
2. 保存文件
3. 复制这两个文件到 HBuilderX 项目中
4. HBuilderX 会自动刷新
5. 浏览器自动显示最新效果
```

### 场景 3：添加了新文件

**例如**：添加了 `components/NewComponent.vue`

```
1. 在源项目中创建新文件
2. 复制新文件到 HBuilderX 项目中
3. 在需要的地方导入使用
4. HBuilderX 会自动刷新
5. 浏览器自动显示最新效果
```

## 💡 最佳实践

### 推荐做法

#### 1. 使用 Git 管理源代码
```bash
# 在源项目目录中
git init
git add .
git commit -m "初始项目"
```

#### 2. 在 HBuilderX 中开发
```
在 HBuilderX 中修改代码
保存文件（Ctrl + S）
浏览器自动刷新
```

#### 3. 定期同步到源项目
```bash
# 当在 HBuilderX 中做了重要修改后
# 复制修改的文件回源项目
# 提交到 Git
git add .
git commit -m "更新说明"
```

## 📊 文件更新检查清单

修改文件后，检查以下内容：

| 文件类型 | 需要复制 | 说明 |
|---------|--------|------|
| `.vue` 组件 | ✅ 是 | 修改后必须复制 |
| `.ts` 文件 | ✅ 是 | 修改后必须复制 |
| `.scss` 文件 | ✅ 是 | 修改后必须复制 |
| `package.json` | ✅ 是 | 添加依赖后必须复制 |
| `pages.json` | ✅ 是 | 修改路由后必须复制 |
| 文档文件 | ❌ 否 | 不需要复制 |
| `node_modules/` | ❌ 否 | 不需要复制 |

## 🔧 快速更新命令

### 使用 Git 查看修改

```bash
# 查看修改了哪些文件
git status

# 查看具体修改内容
git diff

# 查看修改历史
git log --oneline
```

### 快速复制修改的文件

```bash
# 查看最后一次提交修改了哪些文件
git diff HEAD~1 --name-only

# 复制这些文件到 HBuilderX 项目中
```

## 🎯 总结

### 不需要全部复制的原因

1. **只有修改的文件才需要更新**
2. **其他文件保持不变**
3. **节省时间和空间**

### 推荐的更新方式

1. **使用 Git 管理源代码**
2. **在 HBuilderX 中开发**
3. **修改后只复制修改的文件**
4. **定期提交到 Git**

### 快速查看效果

```
修改代码 → 保存（Ctrl + S） → 浏览器自动刷新（1 秒）
```

## 💡 常见问题

### Q: 如何知道哪些文件被修改了？

A: 使用 Git 查看：
```bash
git status
```

### Q: 如何快速复制修改的文件？

A: 使用文件管理器：
1. 打开源项目文件夹
2. 打开 HBuilderX 项目文件夹
3. 并排显示两个窗口
4. 拖拽修改的文件到 HBuilderX 项目中

### Q: 如何回滚到之前的版本？

A: 使用 Git：
```bash
# 查看历史
git log

# 回滚到某个版本
git checkout <commit-hash>
```

### Q: 如果不小心删除了文件怎么办？

A: 使用 Git 恢复：
```bash
# 恢复删除的文件
git checkout -- <file-path>
```
