# 文件复制指南

## 🎯 核心问题

启动项目时，需要复制哪些文件夹到 HBuilderX 项目中？

---

## 📋 需要复制的文件夹

### 必须复制的文件夹（7 个）

```
源项目中的这些文件夹：
├── pages/                    ✅ 必须复制
├── components/              ✅ 必须复制
├── stores/                  ✅ 必须复制
├── services/                ✅ 必须复制
├── mock/                    ✅ 必须复制
├── types/                   ✅ 必须复制
├── styles/                  ✅ 必须复制
└── utils/                   ✅ 必须复制
```

### 不需要复制的文件夹

```
这些文件夹不需要复制：
├── docs/                    ❌ 不需要（文档）
├── node_modules/            ❌ 不需要（依赖）
└── .git/                    ❌ 不需要（版本控制）
```

## 📝 具体步骤

### 第 1 步：打开两个文件管理器窗口

```
1. 打开文件管理器
2. 打开源项目文件夹（当前项目）
3. 再打开一个文件管理器窗口
4. 打开 HBuilderX 项目的 src 文件夹
5. 并排显示两个窗口
```

### 第 2 步：复制文件夹

**在源项目中：**
```
选中这 8 个文件夹：
✓ pages
✓ components
✓ stores
✓ services
✓ mock
✓ types
✓ styles
✓ utils
```

**复制方式：**
```
方式 1：Ctrl + C 复制，然后 Ctrl + V 粘贴
方式 2：右键 → 复制，然后右键 → 粘贴
方式 3：拖拽到另一个窗口
```

### 第 3 步：粘贴到 HBuilderX 项目

```
粘贴位置：HBuilderX 项目的 src 文件夹

粘贴后的结构：
HBuilderX 项目/src/
├── pages/              ✅ 已粘贴
├── components/         ✅ 已粘贴
├── stores/             ✅ 已粘贴
├── services/           ✅ 已粘贴
├── mock/               ✅ 已粘贴
├── types/              ✅ 已粘贴
├── styles/             ✅ 已粘贴
├── utils/              ✅ 已粘贴
├── App.vue
├── main.ts
└── ...
```

### 第 4 步：刷新 HBuilderX

```
在 HBuilderX 中：
按 F5 刷新项目
或者右键项目 → 刷新
```

## 🎯 完整的启动流程

### 第一次启动（完整步骤）

```
1. 在 HBuilderX 中创建 uni-app 项目
   文件 → 新建 → 项目
   选择 uni-app，Vue 3 + TypeScript
   项目名称：user-center

2. 打开两个文件管理器窗口
   窗口 1：打开源项目文件夹
   窗口 2：打开 HBuilderX 项目的 src 文件夹

3. 复制 8 个文件夹
   从源项目复制：pages, components, stores, services, mock, types, styles, utils
   粘贴到：HBuilderX 项目的 src 文件夹

4. 在 HBuilderX 中刷新项目
   按 F5 或右键项目 → 刷新

5. 在 HBuilderX 终端中安装依赖
   菜单 → 工具 → 终端 → 新建终端
   运行：npm install pinia uni-ui

6. 运行项目
   菜单 → 运行 → 运行到浏览器 → Chrome
   浏览器会自动打开，显示项目效果
```

## 💡 常见问题

### Q: 复制到 src 文件夹还是项目根目录？

A: **复制到 src 文件夹**

```
✅ 正确：HBuilderX 项目/src/pages/
❌ 错误：HBuilderX 项目/pages/
```

### Q: 如果 HBuilderX 项目没有 src 文件夹怎么办？

A: 有两种情况：

**情况 1：项目根目录就是 src**
```
HBuilderX 项目/
├── pages/
├── components/
├── App.vue
└── ...
```
此时直接复制到项目根目录。

**情况 2：项目有 src 文件夹**
```
HBuilderX 项目/
├── src/
│   ├── pages/
│   ├── components/
│   └── ...
└── ...
```
此时复制到 src 文件夹。

### Q: 复制后项目还是空白怎么办？

A: 检查以下几点：

```
1. 确保文件夹复制到了正确的位置
2. 在 HBuilderX 中刷新项目（F5）
3. 检查浏览器控制台是否有错误（F12）
4. 确保已经安装了依赖（npm install pinia uni-ui）
5. 重启 HBuilderX
```

### Q: 后续更新时还需要复制所有文件夹吗？

A: **不需要**，只复制修改过的文件夹

```
例如：只修改了 components/UserCard.vue
只需复制 components/ 文件夹
其他文件夹不需要动
```

详见：`docs/02-项目管理/项目更新指南.md`
