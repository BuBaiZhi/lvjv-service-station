## 一、前置环境检查

### 1.1 云开发环境验证

#### 步骤 1：检查云开发集合

在微信开发者工具中：

1. 点击 **云开发** → **数据库**
2. 查看是否存在以下两个集合：
   - ✅ `houses` - 民宿房源（权限：所有用户可读、仅创建者可写）
   - ✅ `posts` - 广场帖子（权限：所有用户可读、仅创建者可写）

如果不存在，需要**手动创建**（见附录）。

#### 步骤 2：检查权限设置

对于 `houses` 和 `posts` 集合，点击设置：

```
权限设置 = "所有用户可读、仅创建者可写"
```

这样可以保证：
- 所有用户都能看到他人的房源和帖子（可读）
- 只有创建者能修改自己的发布（仅创建者可写）

### 1.2 后端环境验证

#### 步骤 1：启动后端服务

```bash
cd backend
npm start
```

检查输出：

```
✅ Server running on http://localhost:3000
✅ Database connected
```

#### 步骤 2：测试后端登录接口

```bash
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"code": "test_code", "userInfo": {"nickName": "测试用户"}}'
```

期望返回：

```json
{
  "accessToken": "...",
  "refreshToken": "...",
  "userInfo": {
    "id": "...",
    "nickname": "测试用户"
  }
}
```

## 二、登录与身份验证

### 2.1 正常登录流程

1. **打开小程序** → 跳转到登录页
2. **点击"使用微信登录"** 
   - ✅ 弹出微信授权框
   - ✅ 授权后显示"是否编辑资料"对话框
   - ✅ 选择"先看看"跳转到首页

### 2.2 验证认证状态

登录成功后，在开发者工具**控制台**查看日志：

```javascript
// 应该看到这些日志
[Login] ✅ 后端登录成功
[Login] 第 5 步：获取云开发 openid...
[Login] 云函数返回 openid: xxxxxxxxx
[Login] ✅ openid 已保存: xxxxxxxxx
[Login] ✅ 认证状态已保存
```

### 2.3 验证本地存储

在开发者工具**Storage**标签页，查看是否有以下数据：

| 键名 | 类型 | 说明 |
|-----|-----|------|
| `authMode` | string | 应为 `"user"` |
| `accessToken` | string | 后端返回的token |
| `refreshToken` | string | 后端返回的token |
| `userId` | string | 当前用户ID |
| `openid` | string | 云开发的openid（最关键！） |
| `userInfo` | JSON | 用户信息对象 |

❌ **如果 `openid` 为空**，后续所有云数据库操作都会失败。

## 三、发布数据准备

### 3.1 发布房源（houses 集合）

1. 进入 **首页** → **发布** → **发布房源**
2. 填写房源信息：
   - 标题：例如 "大理古城边 3室1厅 整租"
   - 位置、价格、描述等
3. 点击**发布**

验证：在云开发**数据库**的 `houses` 集合中应该看到新记录：

```json
{
  "_id": "xxx",
  "_openid": "your_openid",
  "title": "大理古城边 3室1厅 整租",
  "name": "大理古城边 3室1厅 整租",
  "createTime": "2026-02-26T...",
  "price": xxx,
  ...
}
```

### 3.2 发布帖子（posts 集合）

1. 进入 **广场** → **发布按钮** → **发布帖子**
2. 填写帖子信息：
   - 标题：例如 "寻找编程搭子"
   - 类型：选择 "帖子" 或其他类型（活动、技能、分享等）
   - 内容、图片等
3. 点击**发布**

验证：在云开发**数据库**的 `posts` 集合中应该看到新记录：

```json
{
  "_id": "xxx",
  "_openid": "your_openid",
  "title": "寻找编程搭子",
  "type": "post",
  "createTime": "2026-02-26T...",
  ...
}
```

**❌ 关键问题诊断**：

如果发布成功但无法显示：

1. 检查 `_openid` 是否存在且与当前用户的 `openid` 一致
2. 检查 `createTime` 是否为有效的时间戳
3. 检查 `type` 或其他必要字段是否为空

   - 你发布的所有帖子（来自 `posts` 集合）

### 4.2 数据显示验证

#### ✅ 预期行为

**正常情况（已发布数据）**：

| 项目 | 预期结果 |
|-----|---------|
| 列表显示 | 显示2-3条发布内容（房源+帖子混合） |
| 显示类型 | 房源显示"房源" 标签，帖子显示对应的帖子类型标签 |

#### ❌ 常见问题与解决方案

| 症状 | 原因 | 解决方案 |
|-----|-----|---------|
| 页面显示"暂无发布"（有数据） | `openid` 未获取或不正确 | 检查Storage中的openid是否存在 |
| 只显示帖子，不显示房源 | `houses` 集合不存在或无权限 | 在云开发中创建houses集合 |
| 缩图显示404或加载失败 | 图片URL有误 | 检查发布时的图片字段名是否正确 |
| 数据为空但有模拟数据 | 云开发未连接，降级到本地存储 | 检查云开发环境是否正确初始化 |

## 常见错误与解决

### 错误 1：`Unknown column '_openid' in 'field list'`

**原因**：在后端MySQL中查询了 `_openid` 字段（这是云数据库特有的）

**解决**：不要在后端查询 `_openid`，这是云数据库自动管理的系统字段

### 错误 2：`Cannot read property 'eq' of undefined`

**原因**：使用了 `db._.eq()` 方法，该方法在CloudBase中不存在

**解决**：改为直接字符串匹配
```javascript
// ❌ 错误
.where({ _openid: db._.eq(openid) })

// ✅ 正确
.where({ _openid: openid })
```

### 错误 3：`Request uri too long`

**原因**：查询条件过于复杂

**解决**：简化查询，分页加载数据
