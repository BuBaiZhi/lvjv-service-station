# 我的发布功能 - 快速参考卡

## 🚀 快速测试流程（5分钟版）

### 第1步：环境检查（1分钟）
```
□ 打开微信开发者工具
□ 进入 云开发 → 数据库
□ 查看是否有 houses 和 posts 集合
□ 权限应为：所有用户可读、仅创建者可写
```

### 第2步：登录（1分钟）
```
□ 点击"使用微信登录"
□ 授权并选择"先看看"
□ 检查 Storage：应有 openid 字段
□ 开发者工具Console应显示：
  [Login] ✅ openid 已保存: xxxxx
```

### 第3步：准备数据（2分钟）
```
□ 发布 1-2 个房源（首页→发布→发布房源）
□ 发布 1-2 个帖子（广场→发布帖子）
□ 在云开发数据库中确认数据已保存
```

### 第4步：验证功能（1分钟）
```
□ 进入用户中心→我的发布
□ 验证是否显示所有发布（房源+帖子）
□ 点击"房源"标签，应仅显示房源
□ 点击"帖子"标签，应仅显示帖子
```

## ❌ 常见问题快速诊断

| 症状 | 检查项 | 解决方案 |
|-----|--------|---------|
| **显示"暂无发布"** | openid 是否为空 | 重新登录，确认获取 openid |
| **只显示帖子，无房源** | houses 集合是否存在 | 在云开发中创建 houses 集合 |
| **分类筛选无效** | 数据中 type 字段是否正确 | 检查发布时是否设置了正确的 type |
| **缩图加载失败** | 图片 URL 是否有效 | 更换图片来源或使用本地图片 |
| **登录后仍无 openid** | 云函数是否正确返回 openid | 检查云函数 login 的实现 |

## 📊 数据流程图

```
用户登录
  ↓
获取 code + userInfo (wx.getUserProfile + wx.login)
  ↓
调用后端 /api/auth/login
  ↓
后端返回 accessToken + refreshToken
  ↓
调用云函数 login 获取 openid
  ↓
保存 openid 到 Storage
  ↓
跳转首页
  ↓
发布房源/帖子
  ↓
数据自动关联 _openid （云数据库自动添加）
  ↓
进入"我的发布"页面
  ↓
getUserPosts() 查询 _openid = 当前 openid 的所有数据
  ↓
显示房源 + 帖子列表
  ↓
点击分类标签进行筛选
```

## 💡 提示

1. **云数据库权限很重要**：如果权限设为"仅创建者可读"，则其他用户无法查看任何人的发布

2. **_openid 是关键**：所有与当前用户相关的云数据库操作都依赖于正确的 openid，包括：
   - 查询用户发布
   - 获取用户关注
   - 获取用户粉丝
   - 获取用户草稿

3. **调试时启用日志**：
   ```javascript
   // 在 postService.js 中找到这行
   const USE_MOCK = false
   
   // 改为 true 可以切换到模拟数据模式（用于快速测试UI）
   const USE_MOCK = true
   ```

4. **缓存问题**：
   - 小程序会缓存页面和数据
   - 如果数据更新后页面未刷新，可以：
     - 在 onShow() 中重新加载数据
     - 或者清理缓存：`wx.clearStorageSync()`
