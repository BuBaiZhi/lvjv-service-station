# 项目结构对比

## 🎯 核心问题

为什么 HBuilderX 创建的项目没有 `src/` 文件夹？

**答案**：HBuilderX 创建的 uni-app 项目结构与 CLI 创建的项目结构不同。

---

## 📁 两种项目结构对比

### 方式 1：使用 CLI 创建（有 src 文件夹）

```bash
npx degit dcloudio/uni-app-ts user-center
```

**项目结构**：
```
user-center/
├── src/                    ← 有 src 文件夹
│   ├── pages/
│   ├── components/
│   ├── stores/
│   ├── services/
│   ├── mock/
│   ├── types/
│   ├── styles/
│   ├── utils/
│   ├── App.vue
│   ├── main.ts
│   └── ...
├── package.json
├── tsconfig.json
├── vite.config.ts
└── ...
```

---

### 方式 2：使用 HBuilderX 创建（没有 src 文件夹）

```
HBuilderX → 文件 → 新建 → 项目
选择 uni-app，Vue 3 + TypeScript
```

**项目结构**：
```
user-center/
├── pages/                  ← 直接在项目根目录
├── components/
├── stores/
├── services/
├── mock/
├── types/
├── styles/
├── utils/
├── App.vue
├── main.ts
├── pages.json
├── package.json
└── ...
```

---

## ✅ 解决方案

### 如果你用 HBuilderX 创建的项目（没有 src 文件夹）

**复制位置**：直接复制到项目根目录

```
源项目中的文件夹          →    复制到 HBuilderX 项目的根目录
├── pages/               →    项目根目录/pages/
├── components/          →    项目根目录/components/
├── stores/              →    项目根目录/stores/
├── services/            →    项目根目录/services/
├── mock/                →    项目根目录/mock/
├── types/               →    项目根目录/types/
├── styles/              →    项目根目录/styles/
└── utils/               →    项目根目录/utils/
```

**复制后的结构**：
```
HBuilderX 项目/
├── pages/               ✅ 已复制
├── components/          ✅ 已复制
├── stores/              ✅ 已复制
├── services/            ✅ 已复制
├── mock/                ✅ 已复制
├── types/               ✅ 已复制
├── styles/              ✅ 已复制
├── utils/               ✅ 已复制
├── App.vue
├── main.ts
├── pages.json
└── ...
```

## 🔍 如何判断你的项目结构？

### 检查方法 1：查看项目根目录

打开 HBuilderX 项目的根目录，看是否有 `src/` 文件夹：

- ✅ **有 src 文件夹** → 复制到 `src/` 文件夹
- ❌ **没有 src 文件夹** → 复制到项目根目录

### 检查方法 2：查看 App.vue 位置

- ✅ **App.vue 在 src/ 中** → 复制到 `src/` 文件夹
- ❌ **App.vue 在项目根目录** → 复制到项目根目录

## 📋 完整的启动步骤（针对 HBuilderX 创建的项目）

### 第 1 步：在 HBuilderX 中创建项目

```
1. 打开 HBuilderX
2. 文件 → 新建 → 项目
3. 选择 uni-app
4. 选择 Vue 3 + TypeScript 模板
5. 项目名称：user-center
6. 点击创建
```

### 第 2 步：打开两个文件管理器窗口

```
窗口 1：打开源项目文件夹（当前项目）
窗口 2：打开 HBuilderX 项目的根目录
```

### 第 3 步：复制 8 个文件夹

**从源项目中复制**：
```
✓ pages
✓ components
✓ stores
✓ services
✓ mock
✓ types
✓ styles
✓ utils
```

**粘贴到**：HBuilderX 项目的根目录（不是 src 文件夹）

### 第 4 步：刷新 HBuilderX

```
按 F5 或右键项目 → 刷新
```

### 第 5 步：安装依赖

```bash
# 在 HBuilderX 终端中运行
npm install pinia uni-ui
```

### 第 6 步：运行项目

```
运行 → 运行到浏览器 → Chrome
```

## 💡 为什么会有两种结构？

### CLI 创建的项目（有 src 文件夹）
- 使用 Vite 作为构建工具
- 源代码在 `src/` 文件夹中
- 更现代的项目结构
- 适合大型项目

### HBuilderX 创建的项目（没有 src 文件夹）
- 使用 HBuilderX 内置的构建工具
- 源代码直接在项目根目录
- 更简洁的项目结构
- 适合快速开发

## 📞 常见问题

### Q: 我不确定应该复制到哪里怎么办？

A: 查看你的项目根目录：
- 如果看到 `App.vue` 在根目录 → 复制到根目录
- 如果看到 `src/` 文件夹 → 复制到 `src/` 文件夹

### Q: 复制后还是空白怎么办？

A: 检查以下几点：
1. 确保文件夹复制到了正确的位置
2. 在 HBuilderX 中刷新项目（F5）
3. 检查浏览器控制台是否有错误（F12）
4. 确保已经安装了依赖（npm install pinia uni-ui）
5. 重启 HBuilderX

### Q: 为什么我的项目没有 src 文件夹？

A: 因为你用 HBuilderX 创建的项目，这是正常的。HBuilderX 创建的项目结构与 CLI 创建的项目结构不同。

### Q: 我应该用哪种方式创建项目？

A: 推荐使用 **HBuilderX 创建**，因为：
- ✅ 更简洁
- ✅ 开发体验更好
- ✅ 内置调试工具
- ✅ 一键运行到浏览器/小程序

---

**记住**：没有 src 文件夹是正常的，直接复制到项目根目录即可！⚡
